About

We set out to build an intuitive site that collects data about universities, cities, and majors from several sources and presents it to you, a prospective college student, in a way that makes it easier for you to find the information you need to pick the right college. The site has information about all of the universities in the U.S., including key statistics about their admissions and costs for attending. It also has essential information about the most common majors that college students can choose from. Finally, the site contains snapshots of college cities and factors that may influence a prospective student’s decisions about what college to attend.

By aggregating data about universities, majors, and cities, we have centralized much of the key information that students need to choose the best college for them. While university databases and sites that list majors provide helpful information, our site provides information specifically relevant to prospective and current college students.

Our tech stack:

Flask
Our backend framework is flask, a microservices framework for python. Flask is useful in that it is lightweight: it was trivially simple to set up the server and start running it locally. When we want to add new pages, we simply add a route for the new page. We also took advantage of Flask’s template mechanics. Using flask templates, we dynamically generate instance pages for our major, university, and city models. This allows us to reuse the same HTML, instead of adding hundreds of nearly identical HTML files for each university.

Bootstrap
Our frontend framework is bootstrap, which we use alongside HTML and CSS to design our web pages. Bootstrap has been crucial for allowing our pages to be reactive. The pages in our site dynamically resize based on the size of the viewport, allowing our format to remain robust and look clean, despite the window size limitations of the user.

MongoDB & mongoengine
All of our data is dynamically pulled from MongoDB, including the images for every model instance. Our models are defined with MongoEngine, an open-source Python Object-Document Mapper. MongoEngine will allow us to define “schemas” (MongoDB does not technically have schemas since it’s NoSQL) with python objects, so we can interact with our database purely in terms of University, Major, and City python objects that we design.
